# Design Notes

This skill is intentionally narrower than large self-improvement skills.

## What it keeps

- correction capture
- task/project memory
- cross-channel continuity
- session-end summaries
- durable rule promotion into existing workspace files
- proactive internal follow-through for continuity updates

## What it rejects

- separate `~/self-improving/` storage
- automatic promotion/demotion machinery
- large heartbeat-driven rewrites
- duplicate memory systems that compete with existing workspace memory

## Philosophy

The workspace already has a memory model:

- `USER.md`
- `MEMORY.md`
- `memory/YYYY-MM-DD.md`
- `AGENTS.md`
- mission-control task board
- `.learnings/`

Use that model better instead of building a parallel one.

## Proactivity stance

This skill should not just explain where memory belongs.
It should bias the agent toward making the write now when the write is internal, low-risk, and clearly useful.
