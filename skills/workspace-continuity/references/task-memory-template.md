# Task Memory Template

Use this structure for meaningful project/task detail views.

```markdown
Project memory

Goal
- What this task is trying to accomplish

What is known
- Decisions already made
- Constraints
- Relevant context

What has been done
- Concrete work already completed
- Files changed / integrations set up / outcomes verified

Blockers / unknowns
- Missing access
- Unclear decisions
- Dependencies

Next steps
- The next obvious actions
```

## Guidance

- Keep it factual and compact.
- Prefer bullets over long prose.
- Update after real progress, not just at the end.
- If the task was discussed in Discord/Telegram, copy the important substance here.
- If a task becomes obsolete, move it to archive rather than deleting history.
