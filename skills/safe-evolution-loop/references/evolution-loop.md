# Evolution Loop

## 1. Observe

Start from reality, not ambition.

Good sources:
- user corrections
- failed commands
- repeated manual steps
- repeated context loss
- recurring tasks with the same shape
- cross-channel drift between Discord/main/task board

Write down the signal in one sentence.

## 2. Define the target

Turn the signal into a concrete improvement target.

Examples:
- "I keep forgetting to mirror Discord requests into main memory" -> improve cross-channel capture workflow
- "I keep rewriting the same task-detail structure" -> create a task-memory template
- "I misread skill names from ClawHub repeatedly" -> add a routing/vetting rule

## 3. Choose the smallest safe change

Prefer:
- one rule
- one skill update
- one helper script
- one task-board/template change

Avoid large rewrites when a smaller intervention is enough.

## 4. Validate

Use the smallest proof that the change helped:
- command succeeds
- workflow becomes shorter
- failure does not repeat
- context survives across sessions
- task board stays fresher

## 5. Record

Write the result into the right place:
- mission-control task board for project work
- `memory/YYYY-MM-DD.md` for timeline/progress
- `USER.md` for user preference changes
- `AGENTS.md` for future-you rules
- `.learnings/` for unresolved or repeat lessons

## Compact report format

Use this if you need a short write-up:

```markdown
Signal
- What went wrong or kept recurring

Change
- What small thing was changed

Validation
- How it was checked

Outcome
- What improved or what still failed

Next
- What should happen now
```
